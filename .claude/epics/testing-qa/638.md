# Issue #638: Fix CI Workflow Test Python Version Matrix Parsing

## Problem

The test `tests/ci/test_workflows.py::TestCIWorkflow::test_ci_uses_python_312` is failing because it expects a list of Python versions but receives a GitHub Actions expression string.

### Error

```
AssertionError: CI 'test' job must test against exactly 2 Python versions (3.11 and 3.12)
assert 98 == 2
where 98 = len('${{ github.event_name == ... }}')
```

The matrix uses a conditional expression to select different Python versions for PR vs full runs, but the test parser doesn't handle this.

## Solution

Update the test to properly handle GitHub Actions matrix expressions:
1. Detect when matrix value is an expression string
2. Parse/validate the expression
3. Extract version arrays from the expression

### Files to Modify
