# Issue #535: Replace time.sleep with deterministic wait in test_auth_store

## Description

`test_store_refresh_with_zero_ttl_expires_immediately` used `time.sleep(0.01)` for
timing-dependent assertions, making it inherently flaky on slow CI runners.

## File Modified

- `tests/api/test_auth_store.py`

## Resolution

Replaced `time.sleep(0.01)` with a deterministic time freeze using
`unittest.mock.patch` on `file_organizer.api.auth_store.time.time`, ensuring that
the refresh logic is exercised through the public API without relying on real-time
delays and avoiding flakiness on slower CI runners.

Resolved in PR #536.
