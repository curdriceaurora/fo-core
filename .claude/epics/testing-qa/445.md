# Bug #445: Invalid Click Short Option Validation

## Issue
Copilot identified that `@click.option("--force", "-", ...)` uses a single dash (`"-"`) as a short option, which is invalid in Click. Click requires proper short flags like `"-f"` or the short form should be omitted entirely.

## Problem
- Single dash `"-"` is not a valid short option format in Click
- Can raise errors at import/CLI registration time
- Need to audit all Click options for similar issues

## Solution
Replace single dash with valid short flag or remove the short form:
```python
# BEFORE (invalid)
@click.option("--force", "-", ...)

# AFTER (valid)
@click.option("--force", "-f", is_flag=True, ...)
# OR
@click.option("--force", is_flag=True, ...)  # Remove short form if not needed
```

## Files to Check
- Any file with `@click.option` declarations
- Particularly: `src/file_organizer/cli/*.py`
