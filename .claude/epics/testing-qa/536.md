# Task #536: Implement Smoke Test Suite for PR CI Optimization

## Overview

Optimize PR CI feedback time by implementing a focused smoke test suite that runs critical tests while keeping full test runs for main branch validation.

**Current State:**
- PR CI runs all tests except regressions (~10,600 tests, 4-5 minutes)
- Main branch runs full test suite including regressions (~10,600+ tests, 5+ minutes)

**Goal:**
- Create a fast smoke test suite (~500-1000 tests, <1 minute)
- Reduce PR CI feedback time from 4-5 min to ~1 min
- Maintain full regression detection on main branch

## Acceptance Criteria

- [ ] Mark critical test cases with `@pytest.mark.ci` marker
  - Core daemon/service tests (thread safety, signal handling)
  - Core API tests (cache, auth, routes)
  - Core pipeline/orchestrator tests
  - Cache and storage layer tests
  - Target: 500-1000 tests with `ci` marker

- [ ] Update `.github/workflows/ci.yml`
  - Change PR test command from `-m "not regression"` to `-m "ci"`
  - Keep full suite for main branch pushes
  - Verify coverage requirement still met on smoke tests

- [ ] Verify implementation
  - Run smoke test locally: `pytest -m ci --cov=file_organizer --cov-fail-under=95`
  - Confirm coverage meets 95% threshold
  - Confirm all critical paths covered
  - Push and verify CI runs in ~1 minute

## Technical Details

### Current Infrastructure
✅ Pytest marker `ci` already defined in `pyproject.toml`:
```
markers = [
    "ci: marks tests intended for PR CI validation",
    ...
]
```

### Implementation Strategy

1. **Identify critical tests** to mark with `@pytest.mark.ci`:
   - All daemon service tests (thread safety critical)
   - All cache/storage tests (data integrity)
   - Core API endpoint tests
   - Core pipeline/orchestrator tests
   - Auth/security tests
   - File processing core tests
   - Skip: UI, web interface, reader edge cases (tested on full run)

2. **Apply marker** to test classes/functions:
   ```python
   @pytest.mark.ci
   class TestCriticalComponent:
       ...
   ```

3. **Update CI workflow** (`.github/workflows/ci.yml` line 120):
   ```bash
   # Before:
   pytest tests/ -m "not regression" ...

   # After (PR only):
   pytest tests/ -m "ci" ...

   # Main stays full:
   pytest tests/ --cov=file_organizer ...
   ```

4. **Test locally**:
   ```bash
   pytest -m ci -v --cov=file_organizer --cov-fail-under=95
   ```
