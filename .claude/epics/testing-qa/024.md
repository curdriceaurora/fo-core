## Description

The CI workflow test file `tests/ci/test_workflows.py` has several pre-existing mypy type annotation warnings that are currently non-blocking but should be addressed for clean type checking.

## mypy Warnings

Surfaced during pre-commit validation for PR #429:

1. **Missing stub package** (`types-PyYAML` not installed):
   ```
   tests/ci/test_workflows.py:16: error: Library stubs not installed for "yaml"
   Hint: python3 -m pip install types-PyYAML
   ```

2. **Overload mismatch** on dict `.get()` call with boolean key:
   ```
   tests/ci/test_workflows.py:39: error: No overload variant of "get" of "dict"
   matches argument types "bool", "dict[Never, Never]"
   ```

3. **Missing type parameters** for generic `dict` (bare `dict` without `dict[str, Any]`):
   ```
   tests/ci/test_workflows.py:77, 81, 88, 93, 98, 136, 141, 146, 150, 164, 198
   error: Missing type parameters for generic type "dict"
   ```

## Files Affected

- `tests/ci/test_workflows.py`
- `pyproject.toml` (add `types-PyYAML` to dev dependencies)

## Fix Plan

1. Add `types-PyYAML` to `[project.optional-dependencies] dev` in `pyproject.toml`
2. Replace bare `dict` with `dict[str, Any]` throughout `test_workflows.py`
3. Fix the boolean dict key overload at line 39 (use correct type for the `.get()` call)
4. Verify `mypy` passes cleanly on the file
