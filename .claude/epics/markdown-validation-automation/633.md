# Task: Add full markdown linting to pre-commit validation

## Description

Enhance `.claude/scripts/pre-commit-validation.sh` to run full markdown linting using `pymarkdown` (configured in `.pymarkdown.json`). Currently, the script only performs selective markdown checks (broken links, MD022, format conformity) but misses other violations that are caught only in CI (MD041, MD003, MD040, MD013, MD031/MD032, etc.).

This task adds comprehensive markdown linting to the pre-commit validation script, catching all markdown violations locally before pushing to CI.

## Problem

Issue #579 docstring coverage work had markdown linting errors caught only in CI (pymarkdown failures), requiring a fix commit and re-run. These violations could have been caught locally in pre-commit validation, reducing feedback loops.

## Solution

Add full markdown linting to Section 7a-3 of `.claude/scripts/pre-commit-validation.sh`:

```bash
# Section 7a-3: Full markdown linting (NEW)
echo '🔍 Running full markdown linting...'
if command -v pymarkdown >/dev/null 2>&1; then
  if ! pymarkdown scan docs/ .claude/ README.md; then
    echo '❌ Markdown linting failed'
    exit 1
  fi
  echo '✓ Markdown linting passed'
fi
```
