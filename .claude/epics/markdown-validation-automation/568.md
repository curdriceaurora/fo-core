# Task: Fix all existing markdown violations

## Description

Run `pymarkdown scan` across `.claude/` and `docs/` directories, then fix all reported violations so the gates are clean from day one. This is the bulk of the work — many files will need fixes for heading spacing (MD022), trailing whitespace (MD009), and bare code fences (MD040).

## Acceptance Criteria

- [ ] `pymarkdown scan .claude/` reports zero violations
- [ ] `pymarkdown scan docs/` reports zero violations
- [ ] No content or semantic changes — only formatting fixes
- [ ] Frontmatter blocks preserved exactly as-is
- [ ] Code block content unchanged
- [ ] All fixes in a single commit for clean history

## Technical Details

### Expected Violation Types

Based on historical CodeRabbit findings:
- **MD022**: Missing blank lines around headings — most common
- **MD009**: Trailing whitespace
- **MD012**: Multiple consecutive blank lines
- **MD040**: Bare code fences without language annotation
- **MD047**: Missing newline at end of file
- **MD031/MD032**: Blank lines around fenced code blocks / lists

### Approach

1. Run `pymarkdown scan .claude/ docs/` to get full violation list
2. Group violations by type for efficient batch fixing
3. Fix MD022 first (most common) — add blank lines around headings
4. Fix MD009 — strip trailing whitespace
5. Fix MD012 — collapse multiple blank lines to one
6. Fix MD040 — add language annotations to bare code fences (use `text` for generic, `bash` for commands, `python` for Python)
7. Fix MD047 — ensure newline at EOF
8. Re-scan to verify zero violations

### Files Affected

- All `.md` files in `.claude/` directory tree
- All `.md` files in `docs/` directory tree
- Estimated 50-200+ files depending on violation count
