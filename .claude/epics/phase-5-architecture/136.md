# Task: Parallel Processing Implementation

## Description
Implement multi-core parallel processing for batch file organization to achieve the target 3x speed improvement. This includes work distribution across CPU cores, thread/process pool management, and result aggregation for processing large file collections efficiently.

## Objectives
- Implement concurrent file processing using ProcessPoolExecutor and ThreadPoolExecutor
- Create intelligent work distribution based on file type and size
- Build result aggregation and error collection from parallel workers
- Add configurable concurrency limits based on system resources
- Implement progress tracking across parallel workers
- Target 3x speed improvement over sequential processing

## Acceptance Criteria
- [ ] ProcessPoolExecutor for CPU-bound tasks (text extraction, hashing)
- [ ] ThreadPoolExecutor for I/O-bound tasks (file reading, model inference)
- [ ] Configurable worker count (default: CPU count - 1)
- [ ] Work queue with fair distribution across workers
- [ ] Result aggregation preserving processing order
- [ ] Error isolation: one worker failure does not crash others
- [ ] Progress callback for batch progress tracking
- [ ] Memory-aware scheduling: large files processed sequentially if needed
- [ ] Benchmark showing >= 3x improvement on 1000+ file batches
- [ ] Resource monitoring: CPU, memory, disk I/O during processing
- [ ] Unit tests for work distribution and result aggregation
- [ ] Performance benchmarks documented

## Technical Details
- Use `concurrent.futures` for pool management
- File categorization for optimal scheduling: small text files batch together, large media files limit concurrency
- Model inference serialization: Ollama handles one request at a time, so queue model calls
- I/O parallelism: read files in parallel, process sequentially through model
- Chunk large batches into sub-batches for memory management
- Key modules: `services/batch/parallel.py`, `services/batch/scheduler.py`, `services/batch/worker.py`

## Dependencies
- None (parallel start task, foundational for batch optimization)
