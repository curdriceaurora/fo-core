# Task: Database Indexing and Query Optimization

## Description
Optimize database performance for the operation history, undo/redo system, and intelligence services by adding proper indexes, optimizing queries, and implementing connection pooling. As the system scales to handle 100,000+ files, database performance becomes critical.

## Objectives
- Audit and optimize all database queries across the application
- Add indexes for frequently queried columns
- Implement connection pooling for concurrent database access
- Optimize bulk insert and update operations
- Add query performance monitoring and slow query logging
- Target: all queries complete in under 100ms for 100K+ records

## Acceptance Criteria
- [ ] Database schema audit completed with optimization recommendations
- [ ] Indexes added for: file path lookups, timestamp ranges, status filtering
- [ ] Composite indexes for common query patterns
- [ ] Connection pooling via SQLAlchemy pool configuration
- [ ] Bulk operations use batch insert/update (not row-by-row)
- [ ] Query execution plans analyzed for all critical paths
- [ ] Slow query logging: queries > 100ms are logged with warnings
- [ ] VACUUM and ANALYZE scheduled for SQLite maintenance
- [ ] All critical queries benchmarked at 100K records
- [ ] Query performance tests added to test suite
- [ ] Database migration for new indexes

## Technical Details
- SQLAlchemy index definitions on model classes
- Key indexes: `operation_history(file_path)`, `operation_history(timestamp)`, `operation_history(status, timestamp)`, `job_files(job_id, status)`
- Connection pool: `pool_size=5`, `max_overflow=10`, `pool_timeout=30`
- Bulk operations: `session.bulk_insert_mappings()` and `session.bulk_update_mappings()`
- SQLite-specific: WAL mode for concurrent reads, PRAGMA optimizations
- Query profiling via SQLAlchemy event hooks
- Key modules: `history/database_manager.py`, `services/intelligence/` database modules
