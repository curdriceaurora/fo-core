# Task: Background Daemon Mode

## Description
Implement a background daemon mode that runs the file watcher and auto-organization pipeline as a persistent system service. This enables the file organizer to continuously monitor directories and organize files without requiring an active terminal session.

## Objectives
- Create daemon process with proper lifecycle management (start, stop, restart)
- Implement PID file management for process tracking
- Add signal handling for graceful shutdown (SIGTERM, SIGINT)
- Build CLI commands for daemon control
- Implement logging to file for daemon output
- Add system tray integration placeholder for future GUI

## Acceptance Criteria
- [ ] `file-organizer daemon start` starts background process
- [ ] `file-organizer daemon stop` gracefully stops daemon
- [ ] `file-organizer daemon restart` restarts daemon
- [ ] `file-organizer daemon status` shows running/stopped state
- [ ] PID file created at configurable location (default: `~/.file-organizer/daemon.pid`)
- [ ] Log file rotation configured (default: `~/.file-organizer/logs/daemon.log`)
- [ ] Graceful shutdown: finish current file, then stop
- [ ] Signal handling: SIGTERM for stop, SIGHUP for config reload
- [ ] Automatic restart on crash (configurable)
- [ ] Systemd unit file template for Linux
- [ ] launchd plist template for macOS
- [ ] Unit tests for daemon lifecycle management
- [ ] Integration tests for daemon start/stop cycle

## Technical Details
- Use `python-daemon` or custom fork/double-fork daemonization
- PID file management with file locking to prevent duplicate daemons
- Logging via Python `logging` with `RotatingFileHandler`
- CLI commands via Typer integration in `cli/daemon.py`
- Config reload: re-read watch directories and rules without restart
- Key modules: `services/daemon/manager.py`, `services/daemon/process.py`, `cli/daemon.py`

## Dependencies
- Task #133 (File system monitoring with watchdog) for file watching
- Task #134 (Auto-organization pipeline) for processing logic
