# Task: Pub/Sub Event System

## Description
Build a publish/subscribe event system on top of Redis Streams to enable decoupled communication between file organizer components. This layer provides topic-based routing, event filtering, and subscription management so services can react to events without direct coupling.

## Objectives
- Implement topic-based pub/sub abstraction over Redis Streams
- Create event bus for centralized event routing
- Build subscription management with pattern matching
- Add event filtering and transformation capabilities
- Implement event middleware pipeline (logging, validation, metrics)
- Support both sync and async event handlers

## Acceptance Criteria
- [ ] EventBus class with publish/subscribe methods
- [ ] Topic-based routing (e.g., "file.discovered", "file.processed.*")
- [ ] Wildcard pattern matching for subscriptions
- [ ] Event middleware support (pre/post processing hooks)
- [ ] Synchronous and asynchronous handler support
- [ ] Event replay capability for debugging
- [ ] Subscription lifecycle management (subscribe, unsubscribe, pause)
- [ ] Event priority levels (high, normal, low)
- [ ] Backpressure handling for slow consumers
- [ ] Unit tests for event routing and filtering
- [ ] Integration tests with Redis Streams backend

## Technical Details
- EventBus as singleton or dependency-injected service
- Topic hierarchy: `domain.action.detail` (e.g., `file.processed.text`)
- Pattern matching: `file.*` matches all file events, `file.processed.*` matches processed subtypes
- Middleware pipeline: logging -> validation -> metrics -> handler
- Async handlers via `asyncio` integration
- Key modules: `services/events/event_bus.py`, `services/events/middleware.py`, `services/events/subscriptions.py`
