# Task: Automated Releases and Deployment

## Description
Set up automated release pipelines that create versioned releases, publish Python packages, build and push Docker images, and generate changelogs when tags are pushed. This streamlines the release process and ensures every release is reproducible and well-documented.

## Objectives
- Create release workflow triggered by Git tags
- Automate Python package publishing to PyPI
- Automate Docker image building and publishing
- Generate changelogs from commit history
- Create GitHub Releases with artifacts and release notes
- Set up semantic versioning enforcement

## Acceptance Criteria
- [ ] Release workflow triggers on `v*` tag push
- [ ] Python package built and published to PyPI (or TestPyPI)
- [ ] Docker image built, tagged, and pushed to GitHub Container Registry
- [ ] Docker image tagged with: version, latest, git SHA
- [ ] GitHub Release created with auto-generated release notes
- [ ] Changelog generated from conventional commits
- [ ] Multi-platform Docker builds (amd64, arm64)
- [ ] Release artifacts include: wheel, sdist, Docker image manifest
- [ ] Pre-release support for beta/rc versions
- [ ] Dry-run mode for testing release pipeline without publishing
- [ ] Release verification: smoke tests run against published package
- [ ] Documentation updated with release workflow instructions

## Technical Details
- Release workflow: `.github/workflows/release.yml`
- PyPI publishing via `pypa/gh-action-pypi-publish`
- Docker multi-platform builds via `docker/build-push-action` with QEMU
- Changelog generation: `git-cliff` or `conventional-changelog`
- Semantic versioning: `X.Y.Z` with pre-release support (`X.Y.Z-beta.N`)
- GitHub Release: `softprops/action-gh-release` for artifact upload
- Version source of truth: `pyproject.toml` version field
- Key files: `.github/workflows/release.yml`, `cliff.toml` or `.releaserc`, `CHANGELOG.md`
