# Task: Auto-Organization Pipeline

## Description
Build the automatic file organization pipeline that connects file system monitoring events to the existing file processing and organization logic. When new files are detected, they are automatically classified, described, and organized into the appropriate directory structure.

## Objectives
- Create pipeline that routes detected files to appropriate processors
- Integrate with existing TextProcessor and VisionProcessor
- Implement configurable auto-organization rules and policies
- Add conflict resolution for duplicate files and name collisions
- Build notification system for completed and failed organizations
- Support dry-run mode for previewing organization actions

## Acceptance Criteria
- [ ] New files detected by watchdog are automatically queued for processing
- [ ] Files routed to correct processor based on file type (text, image, video, audio)
- [ ] Organization follows configured methodology (PARA, Johnny Decimal, custom)
- [ ] Conflict resolution: skip, rename, overwrite, ask (configurable)
- [ ] Dry-run mode logs what would happen without moving files
- [ ] Notification callbacks for: file organized, file skipped, file error
- [ ] Configurable processing delay after file detection
- [ ] File locking to prevent processing files still being written
- [ ] Pipeline status tracking (queued, processing, completed, failed)
- [ ] Retry logic for transient failures (e.g., model busy)
- [ ] Unit tests for pipeline routing and conflict resolution
- [ ] Integration tests for end-to-end auto-organization

## Technical Details
- Pipeline stages: detect -> validate -> process -> organize -> notify
- File type routing based on extension mapping in existing processors
- File locking via checking file size stability over time
- Processing delay: wait N seconds after last modification before processing
- Integration with existing `FileOrganizer` class as the orchestration entry point
- Key modules: `services/watcher/pipeline.py`, `services/watcher/rules.py`, `services/watcher/notifier.py`
