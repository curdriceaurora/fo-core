# Task: Microservices Communication Layer

## Description

Build the inter-service communication layer that enables file organizer components to operate as independent microservices. This includes service discovery, request/response patterns over events, and circuit breaker logic for resilient communication.

## Objectives

- Implement service registry for microservice discovery
- Build request/response pattern over event streams
- Create circuit breaker for fault tolerance
- Add service health monitoring and heartbeat protocol
- Implement load balancing across service instances
- Design service contracts and API boundaries

## Acceptance Criteria

- [ ] Service registry with registration and discovery
- [ ] Request/response correlation over Redis Streams
- [ ] Circuit breaker with configurable thresholds (open/half-open/closed)
- [ ] Service heartbeat and health check protocol
- [ ] Timeout handling for inter-service calls
- [ ] Retry logic with exponential backoff
- [ ] Service contract definitions for each microservice boundary
- [ ] Load balancing for multiple service instances
- [ ] Graceful degradation when services are unavailable
- [ ] Unit tests for circuit breaker and retry logic
- [ ] Integration tests for service communication

## Technical Details

- Service registry stored in Redis for shared state
- Correlation IDs for tracking request/response pairs
- Circuit breaker states: closed (normal) -> open (failing) -> half-open (testing)
- Heartbeat interval: configurable, default 30 seconds
- Service boundaries: file_discovery, text_processing, vision_processing, organization, history
- Key modules: `services/mesh/registry.py`, `services/mesh/circuit_breaker.py`, `services/mesh/communication.py`
- Technologies: Redis, asyncio, httpx for HTTP fallback

## Dependencies

- Task #129 (Redis Streams integration) for event transport
- Task #130 (Pub/sub event system) for event routing
