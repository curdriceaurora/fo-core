# Task: File System Monitoring with Watchdog

## Description

Implement real-time file system monitoring using the watchdog library to detect new, modified, and deleted files across configured directories. This is the foundation for automatic file organization, enabling the system to react to file changes without manual intervention.

## Objectives

- Integrate watchdog library for cross-platform file system events
- Create configurable directory watch list with include/exclude patterns
- Implement event debouncing to handle rapid successive changes
- Build file change queue for ordered processing
- Add throttling to prevent system overload during bulk operations
- Support recursive and non-recursive directory watching

## Acceptance Criteria

- [ ] Watchdog observer monitors configured directories for file events
- [ ] File events detected: created, modified, moved, deleted
- [ ] Configurable watch directories via config file and CLI
- [ ] Include/exclude patterns using glob syntax (e.g., `*.pdf`, `!*.tmp`)
- [ ] Event debouncing with configurable delay (default: 2 seconds)
- [ ] Throttling: max N files processed per minute (configurable)
- [ ] File change queue with priority ordering
- [ ] Graceful handling of permission errors and inaccessible directories
- [ ] Cross-platform support: macOS (FSEvents), Linux (inotify), Windows (ReadDirectoryChanges)
- [ ] Unit tests for event filtering and debouncing
- [ ] Integration tests with real file system events

## Technical Details

- Use `watchdog >= 3.0.0` for file system event monitoring
- FileSystemEventHandler subclass for custom event processing
- Debounce implementation: collect events, process after quiet period
- Throttle via token bucket or leaky bucket algorithm
- Queue implementation: `asyncio.Queue` or `queue.PriorityQueue`
- Key modules: `services/watcher/observer.py`, `services/watcher/handler.py`, `services/watcher/config.py`

## Dependencies

- None (parallel start task, foundational for real-time watching)
