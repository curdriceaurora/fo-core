# Task: Event Replay and Monitoring

## Description
Implement event replay capabilities and monitoring dashboards for the event-driven architecture. Event replay enables debugging and reprocessing of failed events, while monitoring provides visibility into event flow, processing latency, and system health.

## Objectives
- Build event replay mechanism for reprocessing historical events
- Create monitoring metrics collection for event throughput and latency
- Implement event audit log for compliance and debugging
- Add alerting for event processing failures and backlogs
- Build CLI commands for event inspection and replay

## Acceptance Criteria
- [ ] Event replay by time range (replay events from last N minutes/hours)
- [ ] Event replay by event ID (replay specific events)
- [ ] Event replay with filtering (replay only certain event types)
- [ ] Metrics collection: events/second, processing latency, error rate
- [ ] Event audit log with searchable history
- [ ] Alert thresholds for: consumer lag, error rate, dead letter queue size
- [ ] CLI command: `file-organizer events list` - show recent events
- [ ] CLI command: `file-organizer events replay` - replay events
- [ ] CLI command: `file-organizer events stats` - show event statistics
- [ ] Monitoring data exportable in Prometheus format
- [ ] Unit tests for replay and metrics collection

## Technical Details
- Leverage Redis Streams XRANGE for time-based replay
- Metrics stored in Redis or local SQLite for persistence
- Prometheus-compatible metrics endpoint for external monitoring
- Event audit log with configurable retention period
- CLI integration via Typer commands
- Key modules: `services/events/replay.py`, `services/events/monitor.py`, `cli/events.py`
