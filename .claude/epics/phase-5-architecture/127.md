# Task: Docker Base Image Updates

## Description
Create and configure Docker base images compatible with Python 3.9+ after the syntax conversion and multi-version validation. This includes selecting optimal base images, creating multi-stage Dockerfiles, and ensuring all dependencies install correctly in the container environment.

## Objectives
- Select optimal Docker base images for Python 3.9+ support
- Create multi-stage Dockerfile for minimal production images
- Configure dependency installation within containers
- Set up GPU support via NVIDIA base images
- Test image builds across different base image variants

## Acceptance Criteria
- [ ] Dockerfile created with multi-stage build (builder + runtime)
- [ ] Base image selected for optimal size and compatibility (e.g., python:3.9-slim)
- [ ] All Python dependencies install correctly in container
- [ ] Ollama integration works within Docker environment
- [ ] GPU-enabled variant available (NVIDIA CUDA base)
- [ ] Image size optimized (target: under 2 GB without models)
- [ ] Health check endpoint configured
- [ ] Docker Compose file created for development setup
- [ ] Volume mounts configured for model storage and file access

## Technical Details
- Base images to evaluate: `python:3.9-slim`, `python:3.10-slim`, `python:3.11-slim`
- Multi-stage build: builder stage for compilation, runtime stage for execution
- NVIDIA GPU support via `nvidia/cuda` base images
- Ollama must be accessible (either sidecar container or host network)
- Volume mounts needed for: input files, output directory, model cache, config
- Key files: `Dockerfile`, `docker-compose.yml`, `.dockerignore`
