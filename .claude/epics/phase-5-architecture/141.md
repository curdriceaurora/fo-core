# Task: Model Loading Optimization and Caching

## Description
Optimize AI model loading times and implement a caching layer for inference results to reduce redundant processing. Model loading is a significant bottleneck, and caching previously-seen file results dramatically improves batch processing performance.

## Objectives
- Implement model warm-up and keep-alive strategies
- Build inference result caching with configurable TTL
- Create lazy loading for models (load only when needed)
- Add model memory pooling to reduce allocation overhead
- Implement cache invalidation strategies
- Profile and reduce model initialization time

## Acceptance Criteria
- [ ] Model warm-up: pre-load models on application start (configurable)
- [ ] Model keep-alive: maintain loaded models between requests
- [ ] Inference cache: hash-based lookup for previously processed files
- [ ] Cache storage: SQLite for persistence, in-memory LRU for hot cache
- [ ] Cache TTL: configurable per file type (default: 24 hours)
- [ ] Cache invalidation: file content change (hash mismatch)
- [ ] Lazy loading: models loaded on first use, not at startup
- [ ] Memory pooling: reuse model context between inferences
- [ ] Cache hit rate metrics and reporting
- [ ] Model loading time reduced by >= 50% after warm-up
- [ ] Unit tests for cache logic and invalidation
- [ ] Benchmarks comparing cached vs uncached performance

## Technical Details
- Ollama keep-alive: configure `keep_alive` parameter in API calls
- Inference cache key: SHA-256 of file content + model name + prompt hash
- LRU cache: `functools.lru_cache` or custom implementation with size limits
- Persistent cache: SQLite table (file_hash, model, prompt_hash, result, timestamp)
- Memory pooling: reuse Ollama session context between calls
- Profile with `cProfile` and `memory_profiler` for optimization targets
- Key modules: `services/optimization/model_cache.py`, `services/optimization/inference_cache.py`, `services/optimization/warmup.py`
