# Task: Production Docker Deployment

## Description
Create production-ready Docker deployment configuration including Docker Compose orchestration, environment configuration, volume management, networking, and operational tooling. This makes the file organizer deployable with a single `docker compose up` command.

## Objectives
- Create production Docker Compose configuration with all services
- Configure Ollama as a sidecar container with model pre-loading
- Set up volume mounts for persistent data and file access
- Implement environment-based configuration (dev, staging, production)
- Add operational tooling: logs, health checks, backups
- Publish pre-built images to Docker Hub or GitHub Container Registry

## Acceptance Criteria
- [ ] `docker compose up` starts complete file organizer stack
- [ ] Ollama sidecar with pre-pulled models (configurable)
- [ ] Redis sidecar for event streaming (if event system enabled)
- [ ] Volume mounts: input directory, output directory, model cache, database
- [ ] Environment file (`.env`) for all configuration
- [ ] Health check endpoints for all services
- [ ] Log aggregation to stdout/stderr (Docker standard)
- [ ] Graceful shutdown with proper signal handling
- [ ] GPU passthrough configuration (NVIDIA runtime)
- [ ] Docker network isolation for service communication
- [ ] Build and push scripts for CI/CD integration
- [ ] Documentation: quickstart, configuration, troubleshooting

## Technical Details
- Docker Compose v2 syntax with profiles for optional services
- Ollama container: `ollama/ollama` with model volume mount
- Redis container: `redis:7-alpine` for lightweight event streaming
- Network: bridge network for inter-container communication
- GPU: `deploy.resources.reservations.devices` for NVIDIA GPU access
- Image publishing: GitHub Container Registry (`ghcr.io`)
- Key files: `docker-compose.yml`, `docker-compose.prod.yml`, `.env.example`, `scripts/docker-build.sh`
