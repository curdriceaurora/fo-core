# Task: Memory Management and Profiling

## Description
Implement comprehensive memory management and profiling to ensure the file organizer operates efficiently within constrained environments (Docker containers, low-RAM systems). This includes identifying memory leaks, optimizing memory usage patterns, and adding runtime memory monitoring.

## Objectives
- Profile application memory usage across all processing paths
- Identify and fix memory leaks in long-running operations
- Implement memory-aware processing (adapt behavior to available RAM)
- Add memory usage monitoring and alerting
- Optimize peak memory usage during batch processing
- Document memory requirements for different workload sizes

## Acceptance Criteria
- [ ] Memory profiling report for: text processing, vision processing, batch operations
- [ ] Peak memory usage documented per operation type
- [ ] Memory leaks identified and fixed (verified with repeated processing)
- [ ] Memory-aware batch sizing: reduce batch size when memory is low
- [ ] Garbage collection optimization for long-running daemon mode
- [ ] File processing memory bounded: large files streamed, not loaded entirely
- [ ] Memory usage metrics exposed for monitoring
- [ ] Memory limits configurable via environment variable
- [ ] Application runs stable for 24+ hours without memory growth
- [ ] Unit tests verifying memory cleanup after processing
- [ ] Memory profiling documentation with optimization recommendations

## Technical Details
- Profiling tools: `memory_profiler`, `tracemalloc`, `objgraph`
- Memory monitoring: `psutil.virtual_memory()` checks during processing
- Large file handling: streaming reads instead of full file loading
- Garbage collection: explicit `gc.collect()` after batch operations
- Weak references for cached objects that can be reclaimed
- Context managers ensuring resource cleanup
- Key modules: `services/optimization/memory_monitor.py`, `services/optimization/profiler.py`
