# Task: GitHub Actions Workflows

## Description

Create comprehensive GitHub Actions CI/CD workflows for automated testing, linting, type checking, and build verification on every push and pull request. This ensures code quality gates are enforced automatically and reduces manual review burden.

## Objectives

- Create CI workflow for testing on push and pull request
- Set up multi-version Python testing matrix (3.9-3.12)
- Add linting and type checking workflows
- Configure code coverage reporting and thresholds
- Add security scanning for dependencies
- Set up Docker image build verification

## Acceptance Criteria

- [ ] CI workflow triggers on push to main and all pull requests
- [ ] Test matrix: Python 3.9, 3.10, 3.11, 3.12 on ubuntu-latest
- [ ] Ruff linting check passes before tests run
- [ ] Mypy strict type checking passes
- [ ] Black formatting check passes
- [ ] Test coverage reported and uploaded to Codecov/Coveralls
- [ ] Coverage threshold enforced (minimum 80%)
- [ ] Dependency security scan via `pip-audit` or `safety`
- [ ] Docker build verification (image builds successfully)
- [ ] Workflow caching for pip dependencies (faster runs)
- [ ] Status badges added to README
- [ ] Branch protection rules documented

## Technical Details

- Workflow files in `.github/workflows/`
- Main workflow: `ci.yml` (test + lint + type check)
- Docker workflow: `docker.yml` (build verification)
- Security workflow: `security.yml` (dependency audit)
- Caching: `actions/cache` for pip packages and pre-commit
- Matrix strategy for Python versions and OS (optional: macOS, Windows)
- Ollama not needed in CI (mock models for testing)
- Key files: `.github/workflows/ci.yml`, `.github/workflows/docker.yml`, `.github/workflows/security.yml`

## Dependencies

- None (parallel start task, independent CI/CD setup)
