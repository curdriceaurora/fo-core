# Task: Redis Streams Integration

## Description

Integrate Redis Streams as the backbone for event-driven architecture. This provides a durable, ordered event log that services can produce to and consume from, enabling decoupled communication between file processing components.

## Objectives

- Set up Redis Streams connection and configuration management
- Implement event producer for publishing file processing events
- Implement event consumer groups for reliable message processing
- Create event schema definitions for all file organizer events
- Build connection pooling and reconnection logic
- Add health checks and monitoring for Redis connectivity

## Acceptance Criteria

- [ ] Redis connection manager with configurable host/port/auth
- [ ] Event producer can publish events to named streams
- [ ] Consumer groups created for different processing stages
- [ ] Event schemas defined for: file_discovered, file_processed, file_organized, file_error
- [ ] Connection pooling implemented for concurrent access
- [ ] Automatic reconnection on connection loss
- [ ] Dead letter queue for failed event processing
- [ ] Event serialization/deserialization (JSON)
- [ ] Unit tests for producer and consumer
- [ ] Integration tests with real Redis instance
- [ ] Configuration via environment variables and config files

## Technical Details

- Use `redis-py` library with async support (`aioredis`)
- Redis Streams API: XADD, XREAD, XREADGROUP, XACK, XPENDING
- Consumer groups ensure each event is processed exactly once
- Event schema as Python dataclasses with JSON serialization
- Connection pooling via `redis.ConnectionPool`
- Key modules: `services/events/redis_client.py`, `services/events/producer.py`, `services/events/consumer.py`, `services/events/schemas.py`
- Technologies: Redis 5.0+, redis-py >= 5.0.0

## Dependencies

- None (parallel start task, foundational for event-driven architecture)
