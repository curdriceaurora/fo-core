# Task #335: Add OpenAI API Support

## Description

Community feature request to add OpenAI-compatible API support alongside existing Ollama local models. Would allow users with OpenAI access to use cloud models as an alternative to local inference.

**Requested configuration:**
- `OPENAI_BASE_URL`
- `OPENAI_API_KEY`
- `OPENAI_MODEL`

## Acceptance Criteria

- [ ] OpenAI-compatible provider class extending `BaseModel`
- [ ] Three config inputs: base URL, API key, model name
- [ ] Works with any OpenAI-compatible endpoint (OpenAI, Azure, local proxies)
- [ ] Documented in configuration guide
- [ ] Unit tests for new provider

## Notes

- Extends existing model abstraction in `models/base.py`
- Should support any OpenAI-compatible endpoint, not just OpenAI itself
- Privacy trade-off: cloud models break the privacy-first architecture — document clearly
- Low effort due to existing model abstraction layer
