# Task #011: Design Plugin Architecture

## Description

Design and implement the foundational plugin architecture that enables third-party extensions to File Organizer. This includes creating plugin interface/base classes, lifecycle management, plugin registry and discovery mechanisms, configuration system, and sandboxing/security model.

The architecture should support:
- Dynamic plugin loading and unloading
- Plugin dependency management
- Isolation and security
- Configuration and state management
- Hook/event system for plugin integration points

## Technical Details

### Plugin Interface Design

```python
# file_organizer/plugins/base.py

from abc import ABC, abstractmethod
from typing import Any, Dict, List, Optional
from dataclasses import dataclass
from pathlib import Path

@dataclass
class PluginMetadata:
    """Plugin metadata."""
    name: str
    version: str
    author: str
    description: str
    homepage: Optional[str] = None
    license: str = "MIT"
    dependencies: List[str] = None
    min_organizer_version: str = "2.0.0"
    max_organizer_version: Optional[str] = None

class Plugin(ABC):
    """Base plugin interface."""

    def __init__(self, config: Dict[str, Any]):
        self.config = config
        self.enabled = False

    @abstractmethod
    def get_metadata(self) -> PluginMetadata:
        """Return plugin metadata."""
        pass

    @abstractmethod
    def on_load(self) -> None:
        """Called when plugin is loaded."""
        pass

    @abstractmethod
    def on_enable(self) -> None:
        """Called when plugin is enabled."""
        pass

    @abstractmethod
    def on_disable(self) -> None:
        """Called when plugin is disabled."""
        pass

    @abstractmethod
    def on_unload(self) -> None:
        """Called when plugin is unloaded."""
        pass
```
