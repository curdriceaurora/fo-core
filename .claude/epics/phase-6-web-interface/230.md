### Pydantic Models

**Request Models**
```python
from pydantic import BaseModel, Field
from typing import Optional, List

class ScanRequest(BaseModel):
    input_dir: str = Field(..., description="Directory to scan")
    recursive: bool = Field(default=True)
    file_types: Optional[List[str]] = None

class OrganizeRequest(BaseModel):
    input_dir: str
    output_dir: str
    methodology: str = Field(default="para", regex="^(para|johnny_decimal)$")
    recursive: bool = True
    dry_run: bool = False
    metadata_options: Optional[dict] = None

class MoveFileRequest(BaseModel):
    source: str
    destination: str
    overwrite: bool = False
```

**Response Models**
```python
class FileInfo(BaseModel):
    path: str
    name: str
    size: int
    created: datetime
    modified: datetime
    file_type: str
    mime_type: Optional[str] = None

class FileListResponse(BaseModel):
    files: List[FileInfo]
    total: int
    skip: int
    limit: int

class ScanResponse(BaseModel):
    job_id: str
    files_found: int
    total_size: int
    file_types: dict

class OrganizeResponse(BaseModel):
    job_id: str
    status: str
    files_processed: int
    files_moved: int
    errors: List[str]
```

### Error Handling

```python
from fastapi import HTTPException, status

class APIException(Exception):
    def __init__(self, status_code: int, detail: str, error_code: str):
        self.status_code = status_code
        self.detail = detail
        self.error_code = error_code

@app.exception_handler(APIException)
async def api_exception_handler(request: Request, exc: APIException):
    return JSONResponse(
        status_code=exc.status_code,
        content={
            "error": {
                "code": exc.error_code,
                "message": exc.detail,
                "timestamp": datetime.utcnow().isoformat()
            }
        }
    )
```

## Notes

- Use dependency injection for services (file system, organizer, etc.)
- Keep routers thin - delegate business logic to services
- Use background tasks for operations > 5 seconds
- Return 202 Accepted for async operations with job_id
- Follow REST conventions: GET (read), POST (create), PUT (replace), PATCH (update), DELETE (delete)
- Use HTTP status codes correctly (200, 201, 204, 400, 404, 500, etc.)
- Include rate limiting headers in responses
- Support both JSON and form-data where appropriate
