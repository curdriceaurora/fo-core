# Task 009: Build Settings & Configuration UI

## Description

Create a comprehensive settings and configuration interface that allows users to manage all aspects of the application, including Ollama model configuration, organization methodology rules, user preferences, theme settings, and system options. The interface should be intuitive, organized into logical sections, and provide validation feedback.

- [ ] Settings navigation with sections (General, Models, Organization, Appearance, Advanced)

- [ ] Theme switcher (light/dark mode, custom themes)
- [ ] System settings (cache management, log level, performance)
- [ ] Import/export settings
- [ ] Reset to defaults functionality
- [ ] Real-time validation and feedback

### JavaScript Interactions

```javascript
// Auto-save settings
let saveTimeout;
document.querySelectorAll('.settings-section input, .settings-section select').forEach(el => {
  el.addEventListener('change', () => {
    clearTimeout(saveTimeout);
    saveTimeout = setTimeout(autoSave, 2000);
  });
});

function autoSave() {
  htmx.ajax('POST', '/api/settings/save', {
    values: getFormData(),
    target: '#save-status'
  });
}

// Theme switcher
document.querySelectorAll('.theme-option').forEach(btn => {
  btn.addEventListener('click', () => {
    const theme = btn.dataset.theme;
    document.documentElement.setAttribute('data-theme', theme);
    localStorage.setItem('theme', theme);
  });
});

// Settings search
document.getElementById('settings-search').addEventListener('input', (e) => {
  const query = e.target.value.toLowerCase();
  document.querySelectorAll('.setting-group').forEach(group => {
    const text = group.textContent.toLowerCase();
    group.style.display = text.includes(query) ? 'block' : 'none';
  });
});
```
