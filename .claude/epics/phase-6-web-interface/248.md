#### 3. API Client Example
```python
# Python client example
import requests

class FileOrganizerClient:
    def __init__(self, base_url: str, api_key: str):
        self.base_url = base_url
        self.headers = {"Authorization": f"Bearer {api_key}"}

    def upload_file(self, file_path: str):
        with open(file_path, "rb") as f:
            response = requests.post(
                f"{self.base_url}/api/v1/files/upload",
                files={"file": f},
                headers=self.headers
            )
        return response.json()

    def organize_files(self, methodology: str, file_ids: list):
        response = requests.post(
            f"{self.base_url}/api/v1/organize",
            json={"methodology": methodology, "file_ids": file_ids},
            headers=self.headers
        )
        return response.json()

# Usage
client = FileOrganizerClient("http://localhost:8000", "your-api-key")
file = client.upload_file("document.pdf")
job = client.organize_files("para", [file["id"]])
```

### Troubleshooting Guide Content

#### Common Issues

**Issue: Upload fails with large files**
```
Error: 413 Payload Too Large

Solution:
1. Check MAX_UPLOAD_SIZE in .env
2. Increase nginx client_max_body_size (if using nginx)
3. Update Docker container resource limits
```

**Issue: Organization job stuck in "pending" status**
```
Possible causes:
1. Ollama service not running
2. Database connection lost
3. Worker process crashed

Troubleshooting steps:
1. Check Ollama: curl http://localhost:11434/api/version
2. Check logs: docker-compose logs web
3. Restart services: docker-compose restart
```

**Issue: WebSocket connection drops frequently**
```
Solutions:
1. Check reverse proxy timeout settings
2. Increase WebSocket ping/pong interval
3. Review Redis connection stability
```
