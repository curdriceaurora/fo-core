# Task: Measurement Framework and Baseline Metrics

## Description
After all rulesets are updated, establish a measurement framework to track whether the new rulesets actually reduce reviewer findings in subsequent PRs. Record baseline metrics from the audit and define a protocol for running the classifier on new PRs.

## Acceptance Criteria
- [ ] Create `.claude/epics/657/updates/measurement-framework.md` with:
  - Baseline metrics table (findings per PR by work type, pre-ruleset)
  - Definition of "finding rate": findings / PR for each work type
  - Target improvement: 50% reduction in classified pattern findings
  - Measurement protocol: classify findings on next 5 test PRs + next 3 feature PRs
- [ ] Create a simple classifier prompt template that can be used to classify PR comments:
  - Input: PR number + inline comment text
  - Output: pattern ID + confidence level
  - Save as `.claude/epics/657/updates/classifier-prompt.md`
- [ ] Document which PRs to run the classifier on next (the first 5 PRs after this epic merges)
- [ ] Add a "ruleset effectiveness" tracking table to the epic.md

## Technical Details
- Baseline finding rates (from issue #657):
  - TEST: 634 findings / estimated ~40 test PRs ≈ 15.9 findings/PR
  - FEATURE: 590 findings / estimated ~35 feature PRs ≈ 16.9 findings/PR
  - DOCS: 343 findings / estimated ~20 docs PRs ≈ 17.2 findings/PR
  - CI: 84 findings / estimated ~10 CI PRs ≈ 8.4 findings/PR
- Target post-ruleset rate: ≤8 findings/PR for TEST and FEATURE; ≤9 for DOCS
