# Task: Implement Sidecar Process Manager

## Description

Create `desktop/src/sidecar.rs` that manages the Python backend as a child process: spawning on a random available port, polling the health endpoint until ready, restarting on crash, and clean shutdown.

## Acceptance Criteria

- [ ] `SidecarManager` struct in `desktop/src/sidecar.rs`
- [ ] Finds available port, spawns PyInstaller binary with `--port {PORT}`
- [ ] Polls `/api/v1/health` until backend reports ready (with timeout)
- [ ] Monitors child process, restarts on unexpected exit (max 3 retries)
- [ ] Clean shutdown: sends SIGTERM, waits 5s, then SIGKILL
- [ ] Emits Tauri events for sidecar state changes (starting, ready, crashed, stopped)
- [ ] Rust unit tests with mock HTTP server

## Technical Details

- Use `tauri-plugin-shell` sidecar feature or manual `std::process::Command`
- Port allocation via binding to port 0 and reading assigned port
- Health polling with exponential backoff (100ms, 200ms, 400ms... up to 30s timeout)
- Tauri event emitter for frontend state display
- Create `desktop/src/sidecar.rs`
