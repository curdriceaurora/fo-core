# Task: Initialize Tauri v2 Project

## Description

Create the Tauri v2 project scaffold at `desktop/` with Cargo.toml, tauri.conf.json, and basic main.rs entry point. This is the foundation for the native desktop shell.

## Acceptance Criteria

- [ ] `desktop/` directory with valid Tauri v2 project structure
- [ ] `desktop/Cargo.toml` with tauri v2, serde, tokio dependencies
- [ ] `desktop/tauri.conf.json` configured for webview pointing to localhost
- [ ] `desktop/src/main.rs` with minimal Tauri app that opens a webview
- [ ] `cargo build` succeeds in `desktop/`
- [ ] `cargo test` passes with basic smoke test

## Technical Details

- Use `cargo create-tauri-app` or manual scaffold
- Configure `tauri.conf.json`: window title, size, devUrl pointing to `http://localhost:{PORT}/ui/`
- Add tauri plugins to Cargo.toml: `tauri-plugin-shell`, `tauri-plugin-dialog`, `tauri-plugin-notification`
- Keep main.rs minimal - just app initialization
- Add `.gitignore` for `desktop/target/`

## Dependencies

- [ ] Rust toolchain installed (rustup)
- [ ] Tauri CLI installed (`cargo install tauri-cli`)

## Effort Estimate

- Size: M
- Hours: 6-10
- Parallel: true (no Python dependencies)

## Definition of Done

- [ ] Tauri project builds successfully
- [ ] Webview opens and shows placeholder page
- [ ] Cargo test passes
- [ ] Project structure matches plan
