# Task: Rename PyInstaller Output to Sidecar Convention

## Description

Update build scripts to produce PyInstaller binaries named according to Tauri sidecar convention: `file-organizer-backend-{arch}` where arch is the target triple.

## Acceptance Criteria

- [ ] macOS builds produce `file-organizer-backend-aarch64-apple-darwin` and `file-organizer-backend-x86_64-apple-darwin`
- [ ] Windows builds produce `file-organizer-backend-x86_64-pc-windows-msvc.exe`
- [ ] Linux builds produce `file-organizer-backend-x86_64-unknown-linux-gnu`
- [ ] Sidecar binary runs `--help` and returns exit code 0
- [ ] Sidecar binary starts server on specified port via `--port` flag
- [ ] Build scripts updated: `scripts/build_macos.sh`, `scripts/build_linux.sh`, `scripts/build_windows.ps1`

## Technical Details

- Update PyInstaller `--name` flag in each build script
- Add `--port` CLI argument to API server entry point if not present
- Create `tests/ci/test_build_sidecar.py` for binary validation
- Update `tauri.conf.json` sidecar config to reference these names

## Dependencies

- [ ] None - modifies existing build scripts

- [ ] --port flag works
