# Task: Hardware Profiling — Startup Detection & Model Auto-Selection

## Description
Create `core/hardware_profile.py` that detects hardware capabilities at startup and recommends model sizes and worker counts. Extends existing `optimization/resource_monitor.py` (290 LOC, NVIDIA-only GPU detection). Add a `file-organizer hardware-info` CLI command.

## Acceptance Criteria
- [x] `core/hardware_profile.py` exists with `HardwareProfile` dataclass and `detect_hardware()` function
- [x] Detects GPU type: NVIDIA (nvidia-smi), Apple MPS (sysctl), AMD (rocm-smi), or None
- [x] Reports VRAM, available RAM, CPU core count
- [x] Auto-selects `qwen2.5:3b` on systems with < 8GB RAM
- [x] Auto-selects `qwen2.5:7b` on systems with >= 16GB RAM
- [x] `parallel/config.py` uses detected CPU cores for default worker count
- [x] `file-organizer hardware-info` CLI command prints detected profile
- [x] Startup log line: `INFO: Hardware profile: {gpu_type}, {vram}GB VRAM, {ram}GB RAM, {cores} cores`
- [x] 18 tests in `tests/core/test_hardware_profile.py` covering NVIDIA, Apple MPS, AMD, and no-GPU scenarios (psutil fallback chain)

## Technical Details
- **Current state**: `optimization/resource_monitor.py` has `get_gpu_memory()` (NVIDIA only) and `get_system_memory_total()`
- **Approach**: Extend ResourceMonitor patterns. Use `psutil.cpu_count()`, `psutil.virtual_memory()`, platform-specific GPU detection
- **Apple MPS detection**: `sysctl -n hw.optional.arm64` or check for `/usr/bin/system_profiler SPDisplaysDataType`
- **Model recommendation logic**: RAM-based thresholds with user override via config
- **Integration point**: `parallel/config.py` ParallelConfig default max_workers
