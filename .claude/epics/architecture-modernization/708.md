# Task: Model Lifecycle — Domain Registries & Hot-Swap

## Description
Two sub-deliverables: (a) Split unified model registry into domain-specific registries with domain metadata and validation (text/vision/audio); (b) Add `ModelManager.swap_model()` with drain/pre-warm/rollback semantics. Covers PRD Topics 4 (Hot-Swap) and 7 (Domain Registries).

## Acceptance Criteria

### Domain Registries
- [ ] `models/text_registry.py` exists with `TextModelInfo` (includes `context_window: int`, `max_tokens: int`)
- [ ] `models/vision_registry.py` exists with `VisionModelInfo` (includes `supported_formats: list[str]`, `max_resolution: tuple[int, int]`)
- [ ] `models/audio_registry.py` exists with `AudioModelInfo` (includes `supported_formats: list[str]`, `max_duration_seconds: int`)
- [ ] Unified `registry.py` facade delegates to domain registries
- [ ] `file-organizer models list --type vision` shows domain-specific metadata
- [ ] Tests validate all registered models have required domain fields

### Model Hot-Swap
- [ ] `ModelManager.swap_model(model_type, new_model_id)` method exists with type annotations
- [ ] Test: swap succeeds — new model serves requests after swap
- [ ] Test: swap fails — old model continues serving (rollback)
- [ ] Test: in-flight request completes before old model cleanup
- [ ] Thread-safety: concurrent `generate()` calls during swap don't crash

## Technical Details
- **Current registry**: Single `AVAILABLE_MODELS` list with `ModelInfo` dataclass and string `model_type` field
- **Approach (registries)**: Domain-specific `ModelInfo` subclasses. Unified facade queries all three and merges results
- **Approach (hot-swap)**: Pre-warm new model in background thread -> acquire swap lock -> wait for in-flight drain -> atomic reference swap -> cleanup old model. On failure: release lock, keep old model
- **Thread safety**: Extend existing `threading.Lock` in ModelCache. Use `threading.Event` for drain signaling
