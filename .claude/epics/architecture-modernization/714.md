# Task: Proactive Memory Management — Buffer Pool & Batch Sizer Integration

## Description
Add proactive memory allocation for predictable workloads. Create `BufferPool` with acquire/release semantics. Integrate the existing but unused `batch_sizer.py` into the pipeline. Dynamic pool resizing based on `resource_monitor` feedback.

## Acceptance Criteria
- [ ] `optimization/buffer_pool.py` exists with `BufferPool` class
- [ ] `BufferPool.acquire(size)` returns a pre-allocated buffer; `BufferPool.release(buffer)` returns it to pool
- [ ] Pool pre-allocates N buffers at startup (configurable, default 10 x 1MB)
- [ ] Test: 1000-file synthetic batch with pool uses measurably less GC pressure than without
- [ ] `batch_sizer.py` integrated into `PipelineOrchestrator.process_batch()` (currently unused)
- [ ] Resource monitor triggers pool resize when memory pressure > 85%
- [ ] No memory leak: pool size returns to baseline after batch completes
- [ ] Thread-safe: concurrent acquire/release from multiple workers

## Technical Details
- **Current state**: `optimization/` has 12 files (2,963 LOC) — all reactive. `batch_sizer.py` (275 LOC) exists but is not integrated into any pipeline
- **BufferPool approach**: Pre-allocate `bytearray` buffers. Use `queue.Queue` for thread-safe acquire/release. Track outstanding buffers for leak detection
- **batch_sizer integration**: Wire `BatchSizer.compute_batch_size()` into `PipelineOrchestrator` before `process_batch()` to dynamically size batches based on file sizes and available memory
- **Resize logic**: If `resource_monitor.should_evict()` returns True, shrink pool. If utilization > 90%, grow pool (up to hard limit)
