# Task: Hybrid Retrieval with Embedding Cache — BM25 + Vector Search

## Description
Replace current keyword-only search (tiered scoring, max 10K files) with hybrid retrieval combining BM25 lexical index and vector similarity search with score fusion. Add persistent embedding cache (SQLite). Integrate into copilot for context gathering. Covers PRD Topics 5 (Hybrid Retrieval) and 14 (Embedding Cache).

## Acceptance Criteria
- [ ] `services/search/bm25_index.py` exists — keyword retrieval with TF-IDF scoring
- [ ] `services/search/vector_index.py` exists — embedding-based semantic similarity
- [ ] `services/search/hybrid_retriever.py` exists — score fusion combining BM25 + vector
- [ ] `services/search/embedding_cache.py` exists — persistent SQLite cache for embeddings
- [ ] `file-organizer search "quarterly report" --semantic` returns semantically relevant results
- [ ] BM25 index builds in < 5 seconds for 10,000 files
- [ ] Hybrid mode outperforms keyword-only on a 100-file test corpus (measured recall@10)
- [ ] Embedding cache persists to disk (SQLite) and survives restarts
- [ ] Cache hit rate > 90% on second search of same corpus
- [ ] Copilot suggestions use hybrid retrieval for context gathering
- [ ] Existing keyword search (`file-organizer search "query"`) behavior unchanged (backward compatible)

## Technical Details
- **Current state**: `api/routers/search.py` uses keyword matching with tiered scoring (exact=1.0, stem=0.7, extension=0.5, path=0.3). No vector/embedding infrastructure
- **BM25 approach**: Use `rank-bm25` library or implement Okapi BM25. Index file names, paths, and extracted text content
- **Vector approach**: Use Ollama embedding models (already available). Store embeddings as numpy arrays in SQLite BLOB columns
- **Score fusion**: Reciprocal Rank Fusion (RRF) — `score = sum(1 / (k + rank_i))` across BM25 and vector rankings
- **Embedding cache schema**: `CREATE TABLE embeddings (file_path TEXT PRIMARY KEY, embedding BLOB, model TEXT, updated_at TEXT)`
- **Copilot integration**: `services/copilot/` uses hybrid retriever to find relevant files for conversation context
