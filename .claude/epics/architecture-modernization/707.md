# Task: Benchmarking Suite — CLI Command & Statistical Reporting

## Description
Create `file-organizer benchmark` CLI command with suite selection, warmup runs, statistical reporting (median/p95/p99/stddev/throughput), JSON output for CI regression detection, and hardware profile inclusion. Currently zero benchmark infrastructure exists despite MemoryProfiler (286 LOC) collecting per-function data.

## Acceptance Criteria
- [ ] `cli/benchmark.py` exists with Typer command group
- [ ] `file-organizer benchmark --help` shows available suites and options
- [ ] `file-organizer benchmark --suite text --json` produces valid JSON with fields: `median_ms`, `p95_ms`, `p99_ms`, `stddev_ms`, `throughput_fps`
- [ ] Warmup: first 3 iterations excluded from statistics
- [ ] Results include `hardware_profile` section (from Task 003)
- [ ] `file-organizer benchmark --compare baseline.json` shows delta and regression flag
- [ ] Test: statistical output is correct for a known set of synthetic timings

## Technical Details
- **Current state**: No benchmark CLI command. MemoryProfiler collects data, ParallelProcessor calculates fps but neither aggregate
- **Approach**: New Typer command group. Use `statistics` stdlib for median/stdev. Percentiles via sorted-array method
- **Suites**: text (TextProcessor), vision (VisionProcessor), audio (AudioMetadataExtractor), pipeline (PipelineOrchestrator), e2e (full organize)
- **JSON schema**: `{"suite": str, "hardware_profile": {...}, "results": {"median_ms": float, "p95_ms": float, "p99_ms": float, "stddev_ms": float, "throughput_fps": float, "iterations": int}}`
- **Regression detection**: Compare current vs baseline JSON, flag if p95 > 120% of baseline
