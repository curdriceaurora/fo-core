# Task: Double-Buffered Processing — I/O-Compute Overlap

## Description
Add I/O-compute overlap to the pipeline: prefetch next file(s) while current file's LLM inference runs. Configurable prefetch depth, bounded by memory limiter. Builds on composable pipeline stages from Task 006.

## Acceptance Criteria
- [ ] `PipelineOrchestrator` has `prefetch_depth` parameter (default 2)
- [ ] While LLM inference runs on file N, file N+1 is pre-read and preprocessed
- [ ] While batch N writes to disk, batch N+1 starts loading
- [ ] Test: 10-file batch with prefetch is >= 20% faster than without (measured wall clock, synthetic workload)
- [ ] Test: prefetch doesn't exceed `memory_limiter` threshold
- [ ] Test: error in prefetched file doesn't crash pipeline (graceful skip)
- [ ] `--no-prefetch` CLI flag disables overlap for debugging
- [ ] ParallelProcessor futures correctly bounded with prefetch enabled

## Technical Details
- **Current state**: `PipelineOrchestrator.process_batch()` processes files sequentially. ParallelProcessor has bounded futures (2x workers) but no I/O-compute overlap
- **Approach**: Use `concurrent.futures` with separate I/O executor (ThreadPoolExecutor) and compute executor. Pre-submit I/O reads ahead of compute. Stage pipeline: read(N+2) → preprocess(N+1) → analyze(N) → write(N-1)
- **Memory bound**: Prefetched files held in memory. Limit by `prefetch_depth * max_file_size` vs available RAM
- **Integration**: Uses composable stages from Task 006 — prefetch operates at stage boundaries
