# Task: Composable Pipeline — Stage Extraction & Orchestrator Refactor

## Description
Extract 4 pipeline stages from monolithic `PipelineOrchestrator` (424 LOC). Each stage implements a `PipelineStage` protocol and is independently testable, skippable, and replaceable. Orchestrator composes stages via list.

## Acceptance Criteria
- [ ] `pipeline/stages/preprocessor.py` exists — file validation, metadata extraction
- [ ] `pipeline/stages/analyzer.py` exists — LLM analysis (text/vision/audio)
- [ ] `pipeline/stages/postprocessor.py` exists — file naming, folder assignment
- [ ] `pipeline/stages/writer.py` exists — file move/copy operations
- [ ] Each stage implements `PipelineStage` protocol (from Task 002 interfaces)
- [ ] `PipelineOrchestrator` composes stages via list (not hardcoded method calls)
- [ ] Test: custom pipeline with only preprocessor + writer (skip analyzer) works
- [ ] Test: adding a custom stage doesn't require orchestrator changes
- [ ] Existing behavior unchanged: `file-organizer organize` produces same results
- [ ] Stage execution order configurable via config

## Technical Details
- **Current state**: `pipeline/orchestrator.py` (424 LOC) handles routing, processing, and file operations in one class
- **Approach**: Define `PipelineStage` protocol with `process(context) -> context` signature. Extract each concern into a stage. Orchestrator iterates stage list
- **Stage context**: Dataclass carrying file path, metadata, analysis result, destination path — passed through stages
- **Integration**: Depends on Task 001 (core decomposition) since organizer.py also has processing logic that should align
