# Task: Add markdown linting CI workflow

## Description

Create `.github/workflows/docs-lint.yml` to run `markdownlint-cli2` on all markdown files in CI. This catches MD001 (heading hierarchy), MD022 (blank lines around headings), MD040 (code fence language specifiers), and MD041 (H1 as first line) violations before human review.

## Acceptance Criteria

- [ ] `.github/workflows/docs-lint.yml` created and committed
- [ ] Workflow triggers on `push` and `pull_request` for all branches
- [ ] `DavidAnson/markdownlint-cli2-action@v17` used with glob `**/*.md`
- [ ] Rules MD001, MD022, MD040, MD041 enforced (configure via `.markdownlint.json` or inline)
- [ ] Workflow passes against current codebase (fix any pre-existing violations if needed)
- [ ] CI check appears in GitHub PR checks UI

## Technical Details

```yaml
# .github/workflows/docs-lint.yml
name: Lint Markdown
on: [push, pull_request]
jobs:
  markdownlint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: DavidAnson/markdownlint-cli2-action@v17
        with:
          globs: '**/*.md'
```

Rules configuration via `.markdownlint.json` at repo root (if not already present):
```json
{
  "MD001": true,
  "MD022": true,
  "MD040": true,
  "MD041": true,
  "default": false
}
```

Run locally to verify before pushing:
```bash
npx markdownlint-cli2 "**/*.md"
```
