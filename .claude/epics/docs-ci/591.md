# Task: Add link validation CI workflow

## Description

Create `.github/workflows/docs-link-check.yml` to validate that all relative links in markdown files resolve to real files in the repository. This catches broken `../docs/` or `docs/` path references before merge.

## Acceptance Criteria

- [ ] `.github/workflows/docs-link-check.yml` created and committed
- [ ] Workflow triggers on `push` and `pull_request`
- [ ] Relative links validated against filesystem (external URLs skipped)
- [ ] Workflow passes against current codebase
- [ ] Broken link failures show file and line number in CI output

## Technical Details

Use `lychee-action` (Rust-based, fast, supports ignore patterns):

```yaml
# .github/workflows/docs-link-check.yml
name: Check Markdown Links
on: [push, pull_request]
jobs:
  link-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: lycheeverse/lychee-action@v2
        with:
          args: --no-progress --exclude-external '**/*.md'
          fail: true
```

Key flag: `--exclude-external` skips `http://`/`https://` URLs to avoid flakiness from external site availability. Only relative links are validated.

Alternative if lychee has issues: `gaurav-nelson/github-action-markdown-link-check` with config to skip external links.
