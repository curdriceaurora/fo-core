# Task: Add file path existence test

## Description

Create `tests/docs/test_doc_file_paths.py` to extract file path references from documentation and assert they exist in the repository. Catches content drift where docs reference renamed/moved files (e.g., `core/file_organizer.py` renamed to `core/organizer.py`).

## Technical Details

```python
# tests/docs/test_doc_file_paths.py
import re
from pathlib import Path
import pytest

REPO_ROOT = Path(__file__).parent.parent.parent
DOCS_DIR = REPO_ROOT / "docs"

# Match repo-relative paths like src/foo.py, core/bar.py, tests/baz.py
PATH_PATTERN = re.compile(
    r'`((?:src|core|tests|scripts|\.github)/[^`\s]+\.[a-z]+)`'
)

def get_doc_paths():
    """Extract (doc_file, referenced_path) pairs from all docs."""
    pairs = []
    for md_file in DOCS_DIR.rglob("*.md"):
        text = md_file.read_text()
        for match in PATH_PATTERN.finditer(text):
            pairs.append((md_file.relative_to(REPO_ROOT), match.group(1)))
    return pairs

@pytest.mark.parametrize("doc_file,ref_path", get_doc_paths())
def test_referenced_path_exists(doc_file, ref_path):
    """Paths referenced in docs must exist in the repository."""
    assert (REPO_ROOT / ref_path).exists(), (
        f"{doc_file}: references non-existent path '{ref_path}'"
    )
```

Tune the regex pattern to avoid false positives from code examples that show hypothetical paths. May need an allowlist for intentional examples.
