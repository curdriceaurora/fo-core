# Task: Add new-file template for docs

## Description

Create `docs/_template.md` with correct markdown structure as a reference for authoring new documentation files. Makes the right structure (H1 title, `##` subsections, blank lines around headings, language-tagged code fences) the default starting point.

## Acceptance Criteria

- [ ] `docs/_template.md` created
- [ ] Contains H1 title at top (satisfies MD041)
- [ ] Uses `##` for all subsections — no heading hierarchy jumps (satisfies MD001)
- [ ] Blank lines before and after every heading (satisfies MD022)
- [ ] All code fences include language specifier (satisfies MD040)
- [ ] Includes inline comments explaining each structural requirement
- [ ] Template passes `markdownlint` with MD001/MD022/MD040/MD041 rules

## Technical Details

```markdown
# Document Title

Brief one-sentence description of this document's purpose.

---

## Section One

Content here. Each `##` section is a top-level topic.

## Section Two

Subsections use `###` only when genuinely needed for hierarchy:

### Subsection

Content here.

## Code Examples

Always include a language specifier on code fences:

```bash
# Good: language specified
echo "example"
` ` `

```python
# Good: language specified
print("example")
` ` `

## Reference

Links to related docs.
```

Place in `docs/` root so it's discoverable but excluded from navigation (prefixed with `_`).
