# Task: Fix 11 Naive fromtimestamp() Calls

## Description
Add `tz=UTC` parameter to all 11 instances of `datetime.fromtimestamp()` that currently lack timezone information. This eliminates DTZ006 violations and ensures all file stat timestamps are timezone-aware.

## Acceptance Criteria
- [ ] All 11 `fromtimestamp()` calls include `tz=UTC` parameter
- [ ] Each file imports `UTC` from `datetime`
- [ ] `ruff check src/file_organizer/ --select DTZ006` returns 0 violations
- [ ] All existing tests pass after changes
- [ ] File stat timestamp conversions produce timezone-aware datetimes

## Technical Details
Files and instance counts (8 files, 11 instances):
1. `services/analytics/storage_analyzer.py` — 2 instances
2. `services/deduplication/viewer.py` — 1 instance
3. `services/deduplication/backup.py` — 1 instance
4. `services/deduplication/index.py` — 2 instances
5. `services/copilot/rules/preview.py` — 2 instances
6. `history/tracker.py` — 1 instance
7. `tui/file_browser.py` — 1 instance
8. `cli/dedupe.py` — 1 instance

Pattern:

```python
# BEFORE
from datetime import datetime
mtime = datetime.fromtimestamp(stat.st_mtime)

# AFTER
from datetime import UTC, datetime
mtime = datetime.fromtimestamp(stat.st_mtime, tz=UTC)
```

Note: Some files (deduplication/backup.py, deduplication/index.py) also have `datetime.now()` violations — those are handled by Task 002. Coordinate to avoid merge conflicts on shared files.
