# Task 017: Write comprehensive tests for Phase 2 features

## Objective
Create comprehensive test coverage for all Phase 2 features including copilot mode, TUI components, configuration system, CLI commands, and **Phase 3-5 integration points** to ensure reliability and maintainability.

## Context
Phase 2 introduces significant new features (copilot mode, TUI, enhanced CLI) that must integrate with the large Phase 3-5 backend (audio, dedup, intelligence, daemon, parallel processing, events, undo/redo, analytics). Tests must cover both the new features and their integration seams.

## Requirements

### Unit Tests for Copilot Mode
- Test copilot service initialization and lifecycle
- Test suggestion generation and caching
- Test batch processing logic
- Test error handling and recovery
- Test metrics collection and reporting
- Mock external model dependencies appropriately

### Integration Tests for TUI Components
- Test rich interface rendering and layout
- Test interactive menu navigation
- Test progress bar updates during operations
- Test keyboard shortcuts and input handling
- Test theme application and customization
- Test error display and user feedback
- Test panel switching and state management

### Tests for Configuration System
- Test YAML configuration loading and validation
- Test default value handling
- Test configuration file merging (user + defaults)
- Test environment variable overrides
- Test invalid configuration error handling
- Test configuration schema validation
- Test profile management (save/load/switch)
- **Test delegation to existing Phase 5 module configs** (WatcherConfig, DaemonConfig, ParallelConfig, etc.)

### CLI Command Tests
- Test all CLI commands with various options
- Test argument parsing and validation
- Test help text generation
- Test error messages for invalid inputs
- Test dry-run mode for all commands
- Test interactive vs non-interactive modes
- **Test integration of existing Phase 4 CLI modules** (dedupe, undo, autotag, profile, analytics)

### Phase 3-5 Integration Tests (NEW)
- Test TUI audio file display and metadata panel (Task #30 + Phase 3 audio services)
- Test TUI undo/redo integration (Task #34 + Phase 4 undo system)
- Test TUI analytics dashboard (Task #35 + Phase 4 analytics services)
- Test TUI PARA/JD methodology selectors (Task #33 + Phase 3 methodologies)
- Test CLI dedup/intelligence commands (Task #31 + Phase 4 services)
- Test CLI daemon/pipeline commands (Task #32 + Phase 5 daemon/watcher)
- Test ConfigManager delegation to Phase 5 module configs

### Code Coverage
- Achieve minimum 80% code coverage overall
- Critical paths (copilot, TUI, config) >90%
- Integration seams with Phase 3-5 backends >80%
- Test suite runs in <5 minutes
- No flaky tests

## Deliverables
- Unit test suite for copilot mode components
- Integration test suite for TUI components
- Configuration system test suite
- CLI command test suite
- **Phase 3-5 integration test suite** (new)
- Test fixtures and mock data
- Coverage report showing >80% coverage
