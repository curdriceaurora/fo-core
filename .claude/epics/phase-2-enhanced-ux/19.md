# Task: Migrate to Typer CLI framework

## Description

Create the main Typer CLI application and **integrate the 5 existing CLI command modules** from Phase 4 into a unified subcommand structure. Phase 4 already created individual Typer command modules (`dedupe.py`, `undo_redo.py`, `autotag.py`, `profile.py`, `analytics.py`) in `cli/`, but they lack a main app entry point and several core subcommands are missing. This task wires everything together.

**Key distinction**: This is NOT a from-scratch CLI build. It's an integration + gap-filling task.

## Acceptance Criteria

- [ ] Main Typer app in `cli/main.py` with proper entry point
- [ ] Existing Phase 4 CLI modules integrated as subcommands:
  - `file-organizer dedupe` (from `cli/dedupe.py`)
  - `file-organizer undo` / `file-organizer redo` (from `cli/undo_redo.py`)
  - `file-organizer autotag` (from `cli/autotag.py`)
  - `file-organizer profile` (from `cli/profile.py`)
  - `file-organizer analytics` (from `cli/analytics.py`)
- [ ] New subcommands created:
  - `file-organizer organize` - main file organization
  - `file-organizer preview` - dry-run preview of changes
  - `file-organizer config` - manage configuration (from Task #22)
  - `file-organizer model` - model management (from Task #17)
- [ ] `demo.py` migrated or deprecated in favor of new CLI
- [ ] Help text is clear and comprehensive for all commands
- [ ] Common flags: `--verbose`, `--quiet`, `--dry-run`, `--json`
- [ ] `file-organizer --help` and `fo --help` both work
- [ ] All existing tests updated and passing

## Technical Details

### Existing CLI Modules (Phase 4 - INTEGRATE)

These already exist in `src/file_organizer/cli/`:
- `dedupe.py` - Deduplication scan/resolve commands
- `undo_redo.py` - Undo/redo/history commands
- `autotag.py` - Auto-tagging commands
- `profile.py` - User profile management
- `analytics.py` - Analytics dashboard commands

### New Modules to Create

- `cli/main.py` - Main Typer app, registers all sub-apps
- `cli/organize.py` - `organize` subcommand (migrate from `demo.py`)
- `cli/preview.py` - `preview` subcommand (dry-run mode)
- `cli/config_cmd.py` - `config` subcommand (wraps Task #22 ConfigManager)
- `cli/model_cmd.py` - `model` subcommand (wraps Task #17 model switching)

### Entry Points (already in pyproject.toml)

```toml
[project.scripts]
file-organizer = "file_organizer.cli:app"
fo = "file_organizer.cli:app"
```

### Implementation Approach

1. Create `cli/main.py` with root `app = typer.Typer()`
2. Import and register existing Phase 4 CLI modules as sub-apps
3. Create `organize` command migrating logic from `demo.py`
4. Create `preview` command (dry-run wrapper)
5. Create `config` command delegating to ConfigManager
6. Add common callback for `--verbose`, `--quiet`, `--json` flags
7. Ensure `cli/__init__.py` exports the main `app`

### Key Considerations

- Maintain backward compatibility with any existing scripts using `demo.py`
- Use Typer's built-in validation features
- Consistent error handling across all commands
- Progress indicators via Rich for long-running operations
- Support both interactive and non-interactive modes
