# Task: Integrate Analytics Dashboard in TUI

## Description

Add an analytics dashboard panel to the TUI that visualizes file organization statistics, storage analysis, and usage metrics. Phase 4 built analytics services (`services/analytics/`) and a chart generator (`utils/chart_generator.py`). This task creates a TUI panel that displays those analytics interactively.

## Acceptance Criteria

- [ ] Analytics dashboard panel accessible from TUI main menu
- [ ] Storage breakdown by file type (bar chart / table)
- [ ] Organization history timeline (files organized over time)
- [ ] Duplicate detection summary (count, wasted space)
- [ ] Top folders by size and file count
- [ ] Auto-tagging statistics (tags applied, confidence distribution)
- [ ] Refresh button to update analytics in real-time
- [ ] Export analytics to file (CSV/JSON)

## Technical Details

### Existing Infrastructure (Phase 4 - INTEGRATE)

- `services/analytics/` - Analytics computation services
- `utils/chart_generator.py` - Chart/visualization generation
- `cli/analytics.py` - Existing CLI analytics commands (reference for TUI behavior)
- `history/operation_history.py` - Historical operation data for timeline

### TUI Components to Create

- `tui/panels/analytics_panel.py` - Main analytics dashboard
- `tui/widgets/stat_card.py` - Summary statistic card widget
- `tui/widgets/bar_chart.py` - Terminal bar chart widget (Rich-based)
- `tui/widgets/data_table.py` - Sortable data table for detailed stats

### Integration Points

- Use `services/analytics/` for data computation
- Use `chart_generator.py` for Rich-compatible chart rendering
- Pull history data from `operation_history.py` for timeline
- Connect to `services/deduplication/` for duplicate stats
- Connect to `services/auto_tagging/` for tagging stats
