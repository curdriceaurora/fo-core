# Task: Integrate Undo/Redo and Operation History in TUI

## Description

Add undo/redo controls and operation history browsing to the TUI. Phase 4 built a complete undo/redo system (`undo/` - 5 modules) and operation history tracking (`history/` - 6 modules). This task exposes those capabilities in the TUI so users can review, undo, and redo file organization operations visually.

## Acceptance Criteria

- [ ] Undo/redo keyboard shortcuts in TUI (Ctrl+Z / Ctrl+Y)
- [ ] Operation history panel showing recent operations
- [ ] Operation detail view (files moved, timestamps, confidence scores)
- [ ] Undo confirmation dialog with affected files preview
- [ ] Batch undo support (undo entire organization session)
- [ ] History filtering by date, type, or path
- [ ] Visual indicators for undoable vs. non-undoable operations

## Technical Details

### Existing Infrastructure (Phase 4 - INTEGRATE)

- `undo/undo_manager.py` - `UndoManager` class with undo/redo stack
- `undo/rollback_executor.py` - Executes file rollbacks
- `undo/operation_validator.py` - Validates rollback feasibility
- `undo/history_viewer.py` - History browsing/search
- `undo/conflict_detector.py` - Detects conflicts before undo
- `history/operation_history.py` - Operation tracking with SQLite
- `history/operation_transaction.py` - Transaction management
- `history/history_exporter.py` - Export history data
- `history/database_manager.py` - SQLite database management
- `cli/undo_redo.py` - Existing CLI commands (reference for TUI behavior)

### TUI Components to Create

- `tui/panels/history_panel.py` - Operation history list with scrolling
- `tui/panels/undo_dialog.py` - Undo confirmation with file preview
- `tui/widgets/operation_detail.py` - Detailed operation view

### Integration Points

- Wire Ctrl+Z/Ctrl+Y to `UndoManager.undo()` / `UndoManager.redo()`
- Use `history_viewer.py` for history panel data
- Use `conflict_detector.py` to warn before undo
- Connect to live preview (Task #15) for undo preview
