# Task: Implement CLI model switching with auto-download

## Description

Create a dynamic model switching system that allows users to list available models, switch between text/vision/audio models at runtime, and automatically download missing models with progress tracking. Include performance comparison capabilities.

**Phase 5 context**: The `BaseModel` abstraction (`models/base.py`) and model cache (`optimization/model_cache.py`) already exist. This task builds the user-facing CLI and management layer on top of that foundation.

## Acceptance Criteria

- [ ] CLI command to list all available models with details
- [ ] Dynamic model switching without restarting application
- [ ] Automatic detection of missing models
- [ ] Progress bar/indicator during model downloads
- [ ] Download resume capability for interrupted transfers
- [ ] Model performance comparison metrics display
- [ ] Support for text, vision, and audio model types
- [ ] Validation that switched model is compatible with current operation
- [ ] Integration with existing `optimization/model_cache.py` for efficient loading

## Technical Details

### Existing Infrastructure (Phase 5 - BUILD ON)

- `models/base.py` - `BaseModel` interface, `ModelConfig`, `ModelType`, `DeviceType`
- `models/text_model.py` - `TextModel` (Ollama text generation)
- `models/vision_model.py` - `VisionModel` (vision-language)
- `models/audio_model.py` - `AudioModel` (faster-whisper transcription)
- `optimization/model_cache.py` - Model caching and lazy loading

### New Modules to Create

- `models/model_manager.py` - Orchestrates listing, switching, downloading
- `models/downloader.py` - Download manager with progress tracking and resume
- `models/registry.py` - Model registry with metadata (size, type, capabilities)
- `cli/model_cmd.py` - CLI `model` subcommand (registered in Task #19)

### Implementation Approach

1. Create model registry with metadata (size, type, capabilities, Ollama pull command)
2. Implement model loader leveraging existing `model_cache.py` lazy loading
3. Build download manager with Rich progress bars
4. Add model cache management (list cached, clear cache, disk usage)
5. Create performance benchmarking utilities
6. Wire into CLI as `file-organizer model list|switch|pull|compare|cache`

### Key Considerations

- Model storage location and disk space management
- Network error handling and retry logic
- Model compatibility checks before switching
- Memory management when loading large models
- Coordinate with Task #22 (config system) for default model settings
