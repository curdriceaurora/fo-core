## Description

Design and implement a unified, user-facing YAML configuration system for the File Organizer. Phase 5 created multiple module-specific config classes (`WatcherConfig`, `DaemonConfig`, `ParallelConfig`, `PipelineConfig`, `EventConfig`, `DeployConfig`, plus methodology configs for PARA and Johnny Decimal). This task creates a **unified `ConfigManager`** that wraps those existing configs under a single user-facing YAML file with profile support.

**Key distinction**: This task does NOT replace existing module configs. It creates a top-level user config layer that delegates to them.

## Acceptance Criteria

- [ ] YAML configuration schema designed and documented
- [ ] `ConfigManager` class loads/saves `~/.config/file-organizer/config.yaml`
- [ ] Config merging: built-in defaults + user overrides
- [ ] Input validation with helpful error messages
- [ ] Multiple user profiles (default, work, personal, etc.)
- [ ] Exclusion patterns (files/folders to skip)
- [ ] Delegates to existing Phase 5 module configs (watcher, daemon, parallel, events)
- [ ] CLI `--config` flag to specify alternate config path
- [ ] Unit tests for all configuration operations
- [ ] Documentation with configuration examples

## Technical Details

### Existing Module Configs (Phase 5 - DO NOT REPLACE)

These already exist and should be wrapped, not replaced:
- `watcher/config.py` - `WatcherConfig` (exclusion patterns, debouncing, file types)
- `daemon/config.py` - `DaemonConfig` (watch dirs, output dir, PID, logging)
- `parallel/config.py` - `ParallelConfig` (workers, batch size, resource limits)
- `pipeline/config.py` - `PipelineConfig` (processor routing)
- `events/config.py` - `EventConfig` (Redis streams, pub/sub)
- `deploy/config.py` - `DeployConfig` (Docker, scaling)
- `methodologies/johnny_decimal/config.py` - JD-specific config
- `methodologies/para/config.py` - PARA-specific config

### Configuration Schema

```yaml
version: "2.0"
profiles:
  default:
    scan_paths:
      - ~/Documents
      - ~/Downloads
    exclusions:
      patterns: ["*.tmp", ".git/*", "node_modules/*"]
      folders: [".cache", "__pycache__"]
    models:
      text: "qwen2.5:3b-instruct-q4_K_M"
      vision: "qwen2.5vl:7b-q4_K_M"
    organization:
      methodology: "auto"  # auto | para | johnny_decimal | custom
      create_folders: true
      dry_run: false
      batch_size: 50
    daemon:
      enabled: false
      watch_dirs: []
      debounce_seconds: 2.0
    parallel:
      workers: 4
      max_memory_mb: 2048
    ui:
      theme: auto
      show_previews: true
      verbose: false
```

### Implementation Components

**ConfigManager class** (`config/manager.py`):
- `load_config(path: Path) -> Config`
- `save_config(config: Config, path: Path)`
- `validate_config(config: Config) -> ValidationResult`
- `merge_configs(base: Config, override: Config) -> Config`
- `get_default_config() -> Config`
- `get_profile(name: str) -> ProfileConfig`
- `to_watcher_config() -> WatcherConfig` (delegate to existing)
- `to_daemon_config() -> DaemonConfig` (delegate to existing)
- `to_parallel_config() -> ParallelConfig` (delegate to existing)

**File Locations:**
- Default: `~/.config/file-organizer/config.yaml`
- Alternative: `--config` CLI flag
- Fallback: Built-in defaults

### Error Handling

- Invalid YAML syntax -> clear parsing error with line number
- Missing required fields -> list missing fields with examples
- Invalid values -> suggest valid alternatives
- File not found -> auto-create with defaults
